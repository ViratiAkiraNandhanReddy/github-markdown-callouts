## How to Contribute

There are several ways you can contribute:

- **Report bugs or mistakes** in callout examples or README.  
- **Suggest new callout types or improvements**.  
- **Add example Markdown files** demonstrating usage.  
- **Improve documentation** for clarity, grammar, or structure.  
- **Fix formatting or styling issues** in existing Markdown files.  

> [!TIP]
> No contribution is too small! Every fix, suggestion, or example helps improve this resource.

## Reporting Issues

If you notice an error, a broken example, or want to request a new feature:

1. Go to the [Issues tab](https://github.com/ViratiAkiraNandhanReddy/github-markdown-callouts/issues).  
2. Click **New Issue**.  
3. Choose **Bug Report** or **Feature Request**.  
4. Fill in the template with:
    - **Title:** Clear and descriptive  
    - **Description:** Steps to reproduce, expected vs actual behavior  
    - **Markdown snippet (if applicable)**  

> [!WARNING]
> Please **search existing issues** first to avoid duplicates.

## Suggesting Enhancements

If you have ideas to improve the repository:

1. Open a **Feature Request** issue.  
2. Describe the enhancement clearly:
    - Why it is useful  
    - How it benefits users  
    - Any relevant examples  

> [!TIP]
> You can link to external resources, blogs, or GitHub pages that inspired the enhancement.

## Submitting Pull Requests

We encourage pull requests for improvements, fixes, and examples.  

### Steps:

1. **Fork the repository** to your account.

2. **Clone your fork** locally:

```
git clone https://github.com/ViratiAkiraNandhanReddy/github-markdown-callouts.git
```

3. **Create a new branch** for your changes:

```
git checkout -b feature/my-new-callout
```

4. Make your changes and commit:

```
git commit -m "Add example for TIP callout with emoji"
```

5. Push your branch:

```
git push origin feature/my-new-callout
```

6. Open a **Pull Request** to the main repository.

7. Fill in the PR template:
    - Describe the changes  
    - Explain why they are needed  
    - Reference any related issues  

> [!TIP]
> Keep commits **small and descriptive** for easier review.

## Code & Markdown Style Guidelines
  
- Keep callouts **concise and clear**.  
- One idea per callout.  
- Follow **GitHub Markdown conventions** (headings, lists, code blocks).  
- Ensure all examples **render correctly on GitHub**.  
- Maintain alphabetical or logical order when adding new examples.
